Utilities

Structure- and layout-related classes that do one, generic thing extremely well. Can be naturally mixed with each other and with components -- as long as it makes sense of course.

pad

<div gui-u-pad="medium-h">...</div>

Defines a padding for element. Uses -v and -h suffixes for setting only vertical (top & bottom) or horizontal (left & right) margins.

Components

Context-specific selectors for distinct components of an interface. They are forbidden to mix with each other.

button

<div gui-c-button>...</div>

A simple button component with sizes, hover, active and disabled states.

icon

<div gui-c-icon="big"> <use xlink:href="#icon-sonar"> </div>

Small component styling the SVG icons added with <use>. Just add them in _assets/icons and let grunt-svgstore do its work.

default big big & silver

image

<figure gui-c-image="small medievalbloodandgore"></figure>

For creating a fixed-size image based on unknown-size sources. Supports retina media queries. Images resides in _assets/images/namespace/file.jpg and then go through grunt-responsive-images. You should define them in image/library.css partial file.

progressbar

<div gui-c-progressbar>
    <span gui-u-progressbar-indicator style="width: 50%;">...</span>
</div>

Progress bars consist of a bar and an indicator. Indicator size should be probably managed by JavaScript with setting percentage width. It fills the vertical space on default, but has a short modifier too. There is also a nice is-pending animation.
